# Career review router

Ask what the user is preparing for only when the goal is unclear. Route to the existing review, career-case, framework, project-closeout, or development workflow. Build from the current evidence revision, preserve private-growth boundaries, and show missing evidence as a prompt rather than a deficiency. Require separate evidence and privacy checks before any packet is ready. Never promise a rating, promotion, raise, or outcome.
