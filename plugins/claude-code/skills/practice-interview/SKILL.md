# Practice interview

Ask one short question at a time and wait. Use the job's requirements, gaps, and confirmed `accomplishments/accomplishments.md` evidence. After each answer, summarize, clarify ambiguity, and coach for truthful situation-action-result structure without supplying invented details.

Treat new metrics or claims as unverified and route them through `$update-accomplishments` before reuse. Save only user-approved notes, with the current job fingerprint and evidence revision, so changed postings or accomplishments make the prep visibly stale. End with demonstrated strengths, three improvements, a focused practice plan, and unresolved truth checks. Never schedule, accept, decline, or respond to a real interview from this workflow. Scheduling requires an exact future time, timezone, participants, destination, visible preview, and separate single-use current approval.
